## 2. 今天推荐学习的 AI 工具

**工具名称：** GitHub Copilot  
**它能做什么：** 一个 AI 代码补全助手，在你编写代码时，它会根据上下文自动推荐整行或整段代码，支持 Python、JavaScript、C++ 等多种语言。  
**我这个中德电气自动化学生可以怎么用：**  
- 写 PLC 仿真脚本（Python）、数据处理脚本（读取传感器数据）时，Copilot 能自动补全常见的循环、文件读写等代码。  
- 学习英文技术文档时，Copilot 能帮你快速写出对应的示例代码，减少查语法的时间。  
- 在 VS Code 或 JetBrains IDE 中集成，边写边学。  
**今天可以尝试的一个小操作：** 打开 VS Code，在 Python 文件中输入 import csv 后按回车，再输入 # 读取 sensor.csv 并打印每一行，然后按 Tab，看 Copilot 是否自动生成完整代码。  
**来源：** [GitHub Copilot Docs](https://docs.github.com/en/copilot)

## 3. 今天的 Codex 使用技巧

**技巧名称：** 用自然语言注释驱动 Codex 生成函数  
**使用场景：** 当你需要写一个具体功能的代码，但记不清语法或步骤时，可以直接在注释中描述功能。  
**示例提示词：**  
在 Python 文件中输入：
```python
# 写一个函数，接收一个整数列表，返回所有偶数的平方和
```
然后按回车，Codex（或 Copilot）会自动补全函数实现。  
**注意事项：**  
- 注释要用英文或清晰的中文，越具体越好。  
- 检查生成的代码逻辑是否正确，尤其涉及边界条件时。  
- Codex 可能生成过时或不安全的代码，养成阅读和测试的习惯。

## 4. 今天的英文资料阅读

**英文原句：**  
*“In the following examples, input and output are distinguished by the presence or absence of prompts (>>> and …): to repeat the example, you must type everything after the prompt, when the prompt appears; lines that do not begin with a prompt are output from the interpreter.”*  
**中文解释：**  
在下面的例子中，输入和输出通过是否有提示符（>>> 和 …）来区分：要重复示例，你必须在提示符出现后输入提示符之后的所有内容；不以提示符开头的行是解释器的输出。  
**关键词：** prompted（带提示符的）、interpreter（解释器）、distinguished（区分）  
**我应该怎么读懂它：**  
- 先读句子主干：input and output are distinguished by prompts。  
- 遇到生词（如 distinguish）可以查字典，再结合上下文。  
- 这是 Python 官方文档的入门部分，多读几遍就能熟练理解交互式环境的工作方式。  
**来源：** [Python Documentation](https://docs.python.org/3/tutorial/introduction.html)

## 6. 今天可以做的一个小任务

**任务：** 用 Python + GitHub Copilot 写一个简单的传感器数据分析脚本（15-30 分钟）。  
**操作步骤：**  
1. 在 VS Code 中创建一个新的 Python 文件 `analyze_sensor.py`。  
2. 用 Copilot 生成一段代码：在文件开头写 `# 模拟读取三个温度传感器的数值，计算平均值和最大值`，然后按 Tab 让 Copilot 补全。  
3. 手动添加一个简单的数据列表，比如 `temperatures = [25.3, 26.1, 24.8]`。  
4. 运行脚本，确认输出正确。  
5. 小扩展：将结果保存到 CSV 文件（提示 Copilot：`# 将结果写入 result.csv`）。  
**完成标准：** 脚本能运行并打印/保存分析结果。
