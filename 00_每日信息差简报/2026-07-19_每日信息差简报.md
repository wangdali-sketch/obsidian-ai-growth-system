## 3. 今天的 Codex 使用技巧

技巧名称：**用 Codex 生成自动化测试桩（Mock）**  
使用场景：你在写一个 PLC 通信模块的 Python 接口，但实际硬件还没到。这时可以先用 Codex 生成一个模拟外部设备的桩代码，方便你调试主程序。  
示例提示词：  
```
帮我写一个 Python 类，模拟西门子 S7-1200 PLC 的读取和写入操作。类名 MockS7。要有 read_register(address) 和 write_register(address, value) 两个方法。read_register 返回随机整数，write_register 打印写入的内容。包含类型提示和 docstring。
```  
注意事项：  
- 明确告诉 Codex 你想“模拟”什么，以及期望的方法名和返回值类型。  
- 生成的桩代码要检查是否符合实际协议的范围限制，避免超范围值导致后续误判。

## 4. 今天的英文资料阅读

英文原句：  
> In the following examples, input and output are distinguished by the presence or absence of prompts (>>> and …): to repeat the example, you must type everything after the prompt, when the prompt appears; lines that do not begin with a prompt are output from the interpreter.

中文解释：  
在接下来的示例中，输入和输出通过提示符（>>> 和 …）的有无来区分：要重复示例，你必须在提示符出现后输入提示符后面的所有内容；不以提示符开头的行是解释器的输出。

关键词：  
- prompt（提示符）  
- input / output（输入/输出）  
- interpreter（解释器）

我应该怎么读懂它：  
这段文字来自 Python 官方教程的“非正式介绍”。它告诉你如何读懂教程里的代码块：看到 >>> 就是你要敲的命令，没 >>> 的就是运行结果。你完全不用背，只需打开 Python 交互环境跟着敲一遍就能理解。  
**来源：** [Python Documentation](https://docs.python.org/3/tutorial/introduction.html)
