- **它能做什么：** 它像一个在你身边随时待命的编程助手。当你写代码时，它会根据你写的注释和代码上下文，自动推荐下一行代码、整个函数甚至完整的模块。
- **我这个中德电气自动化学生可以怎么用：**

## 4. 今天的英文资料阅读

**英文原句：**
> In the following examples, input and output are distinguished by the presence or absence of prompts (>>> and …): to repeat the example, you must type everything after the prompt, when the prompt appears; lines that do not begin with a prompt are output from the interpreter.

**中文解释：**
在本节的这些例子中，输入和输出通过有没有“>>>”和“...”这些提示符来区分。如果你想亲自试一下例子，就必须在提示符出现时，把提示符后面的所有内容都自己打一遍。行头没有提示符的内容，则是 Python 解释器的输出结果。

**关键词：**
- prompts: 提示符
- distinguished: 被区分
- interpreter: 解释器

**我应该怎么读懂它：**
- **注意动词：** “distinguished by” 是“由……区分”。记住它看到“>>>”就是让你输入代码，看到没有的就是电脑的回应。
- **动手复现：** 按它说的去做，把英文例子自己在本地 Python 环境中敲一遍。不要复制粘贴，手动敲可以帮你记住语法和缩进。
- **工具辅助：** 打开文档的时候，打开 DeepL 或浏览器翻译扩展。遇到完全读不懂的句子，先用翻译看中文意思，再对照英文去理解原文结构。
- **来源：** [Python Documentation](https://docs.python.org/3/tutorial/introduction.html)

## 6. 今天可以做的一个小任务

**任务：在 VS Code 中测试 GitHub Copilot 的自动化代码生成能力**
1. 打开 VS Code，安装 GitHub Copilot 扩展并登录 GitHub 账号（学生可申请免费使用）。
2. 新建一个 Python 文件 `plc_simulator_test.py`。
3. 在文件开头写一段英文注释（不用太复杂）：
   ```
   # Simulate reading a value from a Siemens PLC every 2 seconds for 10 seconds.
   # If the value is greater than 100, print "Alarm: Overload".
   ```
4. 从新的一行开始，按回车并看 Copilot 给你推荐的补全代码。它可能会先推荐 `import time` 之类的语句。
5. 接受它推荐的代码（用Tab键）。
6. **任务验收：** 只需运行一次，看看它自动生成的代码是否能实现注释里的功能。不能也没关系，关键是感受这种“写意图，不写过程”的编程方式。
- **耗时：** 15-20分钟
